- **Microsoft 365 credentials, access tokens, or tenant IDs**

- **Internal business processes or sensitive organizational data**

## Security Best Practices

### Secret Management

- ✅ Use `.env` files for local development (already in `.gitignore`)
- ✅ Use Azure Key Vault for production secrets
- ✅ Use environment variables for configuration
- ✅ Never commit credentials or API keys
- ✅ Review all commits before pushing

### Data Privacy

- All example data in this repository is fictional
- No real customer information is included
- No production system configurations are exposed
- All credentials must be managed through secure channels
